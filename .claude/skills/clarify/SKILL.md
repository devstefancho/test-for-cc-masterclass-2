# Clarify

현재 디렉토리의 `spec.md`를 읽고 아래 4가지 항목을 평가한다.

## 평가 항목 (각 10점, 총 40점)

### 1. 작업 가능성 (10점)
spec 내용만으로 구현에 충분한지 확인한다.
- 기능 요구사항이 구체적으로 기술되어 있는가
- 입력/출력 형식이 명확한가
- 외부 의존성(API, 라이브러리 등)이 명시되어 있는가
- 모호하거나 빠진 요구사항이 없는가

### 2. 보안 이슈 (10점)
보안 취약점 가능성을 확인한다.
- 인증/인가 요구사항이 언급되어 있는가
- 민감 데이터(비밀번호, 토큰 등) 처리 방식이 명시되어 있는가
- 입력값 검증 요구사항이 있는가
- SQL injection, XSS 등 OWASP 주요 취약점 대응이 고려되어 있는가

### 3. TDD 설계 (10점)
테스트 주도 개발이 고려되어 있는지 확인한다.
- 테스트 케이스 또는 테스트 시나리오가 기술되어 있는가
- 엣지 케이스가 명시되어 있는가
- 예상 동작(expected behavior)이 구체적으로 기술되어 있는가

### 4. 명세 분리도 (10점)
spec이 적절히 분리되어 있는지 확인한다.
- 하나의 spec 파일에 의존관계 없는 여러 기능이 뭉쳐져 있지 않은가
- 각 기능이 독립적으로 구현/테스트 가능한 단위로 나뉘어져 있는가
- 기능 간 경계가 명확한가

## 처리 방식

1. `spec.md` 파일을 Read한다
2. 각 항목을 분석하여 0~10점으로 채점한다
3. 항목별 점수와 근거를 출력한다
4. 총점을 계산한다
5. **총점이 30점 미만이면** 다음 경고를 출력한다:

```
⚠️  경고: spec 품질 점수가 {총점}/40점으로 기준(30점)에 미달합니다.
/implement 실행 전에 spec.md를 보완하는 것을 권장합니다.
```

## 출력 형식

```
## Spec 검증 결과

| 항목 | 점수 | 근거 |
|------|------|------|
| 작업 가능성 | X/10 | ... |
| 보안 이슈 | X/10 | ... |
| TDD 설계 | X/10 | ... |
| 명세 분리도 | X/10 | ... |
| **총점** | **X/40** | |

(30점 미만인 경우 경고 메시지 출력)
```
